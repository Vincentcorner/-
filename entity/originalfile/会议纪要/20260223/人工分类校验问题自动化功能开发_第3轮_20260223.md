## 一、会议背景

用户提出对意图匹配 Web UI 的结果展示区域进行功能增强，主要目标是：

1. 在结果表格上方添加"人工分类校验问题自动化"标题
2. 展示自动化校验流程示意图，帮助使用者理解诊断逻辑
3. 为每行结果添加"详情"按钮，点击弹窗展示完整的诊断分析和匹配明细

用户提供了一张流程图截图，描述了当算法意图与标杆意图不一致时的逐步排查逻辑。

## 二、讨论议题与结论

### 2.1 诊断分析流程设计

**讨论内容**：

- 如何自动判定"算法意图与标杆意图不一致"的原因？
- 使用 LLM 还是规则逻辑来生成诊断分析？

**结论**：

采用**规则逻辑（非 LLM 调用）** 实现诊断分析，避免增加 API 成本和延迟。按用户流程图设计 4 步判定链路：

```
算法 ≠ 标杆？
  ├── 标杆在候选列表中？ ──→ 是 ──→ 大模型筛选问题
  │                        └ 否 ↓
  ├── 标杆在权重词表中？ ──→ 否 ──→ 权重分表意图不全
  │                        └ 是 ↓
  ├── 计算标杆意图权重分
  │   ├── 得分 > 0？    ──→ 否 ──→ 特征词匹配不足
  │   └── 得分 > 0      ──→ 是 ──→ topK意图取得不全
```

**5 种诊断结果类型**：

| 类型 | 含义 | 建议措施 |
| ---- | ---- | -------- |
| ✓ 一致 | 算法意图与标杆意图匹配 | 无需处理 |
| 大模型筛选问题 | 标杆意图在候选列表中但 AI 未选中 | 优化意图筛选提示词 |
| 权重分表意图不全 | 标杆意图不在权重词表的意图映射中 | 补充权重词表中的意图映射 |
| topK意图取得不全 | 标杆意图有得分但未进入 Top 候选列表 | 调大 `top_k` 参数 |
| 特征词匹配不足 | 标杆意图在词表中但未命中特征词 | 补充转写策略或调整特征词 |

---

### 2.2 详情弹窗内容设计

**讨论内容**：

- 点击"详情"按钮后弹窗应展示哪些信息？

**结论**：

弹窗分为 4 个区块：

1. **诊断分析区**：醒目样式展示诊断结果（一致=绿色，不一致=橙色警告）
2. **基本信息区**：完整原始问题、转写结果全文、算法意图/标杆意图对比、置信分/转写来源/结果
3. **意图得分明细表**：所有候选意图的得分、命中词数和命中特征词标签
4. **补充信息区**：意图特征词、全量意图列表

---

### 2.3 数据传输方案

**讨论内容**：

- 后端如何向前端传递详情数据？

**结论**：

在 result record 中新增 `详情` 字段（JSON 对象），通过现有 SSE 通道实时推送，包含：

- `原始问题_完整`：不截断的原始文本
- `转写结果`：三维度转写全文
- `全量意图得分`：所有候选意图的得分和命中词数列表
- `全量命中词`：各意图命中的具体特征词
- `诊断分析`：诊断结果文本

前端用 `allResultDetails` 数组存储所有结果，通过索引关联详情按钮与弹窗内容。
