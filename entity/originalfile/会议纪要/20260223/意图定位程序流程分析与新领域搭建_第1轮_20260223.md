### 2.2 切词与权重计算的关系

**讨论内容**：

- 切词和权重分计算是否为两个独立逻辑？
- 各自的输入输出是什么？
- 具体代码实现逻辑？

**结论**：

1. **是两个完全独立的模块**，串行执行：先切词，再计算权重
   - 切词：`QuerySegmenter.segment()` — 基于 AC 自动机在文本中发现命中的特征词
   - 权重计算：`WeightCalculator.calculate()` — 对命中词按公式 `Σ(层级权重 × 词权重)` 计算意图得分

2. **输入输出**：
   - 切词输入：一段文本（str），输出：`[{词, 意图, 层级, 权重}, ...]`
   - 权重输入：切词的输出，输出：`[{意图, 得分, 命中详情}, ...]`（按得分降序，阈值 ≥ 0.4）

3. **切词核心逻辑**：
   - AC 自动机一次线性扫描 → 子串过滤（保留最长匹配）→ 展开为多意图记录 → 负面清单过滤

4. **权重计算核心逻辑**：
   - 层级权重：L1=1.0, L2=0.8, L3=0.6
   - 按意图累加各命中词得分 → 阈值过滤（≥ 0.4）→ 降序排列取 Top-10

### 2.3 领域隔离机制

**讨论内容**：

- 意图分析是针对单领域还是全领域？
- 新领域如何冷/热启动？

**结论**：

1. **当前是单领域分析**。所有数据按领域完全隔离：
   - 原始数据：`originalfile/{领域}/`
   - 权重词表：`result/{领域}/weighted/weighted_words.json`
   - 匹配结果：`result/{领域}/benchmark_compare/`
   - AC 自动机加载哪个领域的词表，就只能匹配到该领域的意图

2. **冷启动路径**（从零开始）：

   ```
   法规文件 → /extract → /intent → /feature-extract → /feature-weight → weighted_words.json → /intent-match
   ```

3. **热启动路径**（有部分数据）：
   - 有意图+改写后问题 Excel → 直接用 `/wordlist` 一步生成 `weighted_words.json`
   - 已有 `weighted_words.json` → 直接跑 `/intent-match`
