## 二、讨论议题与结论

### 2.1 Web UI 页面访问与使用

**讨论内容**：

- 如何启动和访问可视化页面？
- 页面上有哪些功能和配置项？

**结论**：

1. 启动命令：`py scripts/web_app.py`，访问地址 `http://localhost:5000`
2. 页面名称「🎯 意图匹配分析平台」，提供以下配置项：
   - 上传 Excel 文件（拖拽或点击）
   - 领域选择（自动检测 `result/` 目录下的领域）
   - 起始行号（新增功能，默认 1）
   - 分析条数（默认 20，最大 500）
   - 权重词表（自动检测或手动上传）

---

### 2.2 完整功能流程

**结论**：

Web UI 与命令行共享同一套核心逻辑，流程为 5 步：

| 步骤 | 说明 |
|------|------|
| ① 数据配置 | 上传 Excel + 选择领域/条数/词表 |
| ② 三维度转写 | 逐条调用千问 API 转写为情形/群众语言/官方表达 |
| ③ AC 匹配 | 三维度分别独立 AC 匹配 + 加权得分，合并去重 |
| ④ AI 筛选 | 候选 ≥ 2 时调用大模型选出最终意图 |
| ⑤ 结果输出 | 实时展示 + Excel 下载 |

---

### 2.3 意图筛选规则与置信分判定

**讨论内容**：

- 候选意图的筛选规则和阈值是什么？
- 高置信分的判定依据是什么？
- 是所有超过阈值的意图都会获取到，还是有其他规则？

**结论**：

1. **筛选链路**：

   ```
   所有意图 → 得分 ≥ 0.4 过滤 → 按分数降序 → 取前5个（top_k）→ 若≥2个则AI筛选出1个
   ```

2. **阈值配置**（`config/threshold.json`）：
   - `min_score = 0.4`：意图加权得分的最低阈值
   - `top_k = 5`：最多保留的候选意图数量
   - 代码硬上限 `MAX_RESULTS = 10`，实取 `min(top_k, MAX_RESULTS)`

3. **得分计算**：`意图得分 = Σ(层级权重 × 词自身权重)`
   - L1（事项词）权重 1.0，L2（动作词）0.8，L3（情境词）0.6

4. **置信分来源**：

   | 场景 | 公式 |
   |------|------|
   | 无匹配 | 0 |
   | AI 筛选 | 大模型返回的置信度 × 100（上限 99） |
   | 算法 Top-1 | min(80 + 得分 × 10, 99) |

5. **页面颜色分级**：🟢 ≥ 80 / 🟡 ≥ 50 / 🔴 < 50

### 2.4 领域配置 `domains.json` 的作用

**讨论内容**：

- `config/domains.json` 配置文件在系统中的实际作用是什么？

**结论**：

经代码搜索验证，`domains.json` 在当前意图匹配主流程（`web_app.py`、`batch_intent_match.py`）中**未被实际使用**。Web UI 的领域列表实际通过扫描 `result/` 目录下的子文件夹来获取。该配置为 `ConfigManager` 的预留通用能力。

## 三、功能优化

### 3.1 新增"起始行号"输入框

**需求**：支持在页面上直接指定从 Excel 第几行开始处理，无需拆分文件即可分批测试。

**实现**：

- 前端（`index.html`）：在领域和分析条数之间新增"起始行号"输入框，默认值 1（1-indexed）
- 后端（`web_app.py`）：接收 `start` 参数，使用 `df.iloc[start:start+count]` 替代 `df.head(count)`

**分批测试示例**：

| 批次 | 起始行号 | 分析条数 | 处理范围 |
|------|---------|---------|---------|
| 第 1 批 | 1 | 20 | 第 1~20 条 |
| 第 2 批 | 21 | 20 | 第 21~40 条 |
| 第 3 批 | 41 | 20 | 第 41~60 条 |

### 3.2 输出文件命名优化

**需求**：通过文件名即可识别处理了哪个文件的哪些数据。

**实现**：

- 命名规则：`{源Excel文件名}_R{起始行号}_C{实际条数}`
- 示例：上传 `深圳原始数据（部分）.xlsx`，起始行号 21，分析 20 条 → `深圳原始数据（部分）_R21_C20.xlsx`
- 下载时也使用动态文件名
