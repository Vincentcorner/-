# 会议纪要：诊断分析逻辑缺陷修复

> **会议日期**：2026-02-23  
> **参会人员**：用户、AI助手  
> **会议主题**：修复 `diagnose_mismatch` 诊断分析的意图名称匹配缺陷  
> **轮次**：第4轮  

---

## 一、会议背景

用户在使用第3轮新增的"详情分析"功能时发现：标杆意图「了解失业登记」在权重词表中明确有特征词「失业登记」，且意图得分明细中「办理失业登记」已命中该特征词（得分0.96），但诊断分析却给出了错误结论 ——「特征词匹配不足，得分为0」。

---

## 二、问题分析

### 2.1 用户反馈的具体 Case

| 字段 | 值 |
| ---- | ---- |
| 原始问题 | 了解失业登记 |
| 标杆意图 | 了解失业登记 |
| 算法意图 | 查询失业保险待遇申领办理渠道清单 |
| 候选意图 #1 | 办理失业登记（得分0.96，命中词：失业登记） |
| 候选意图 #2 | 查询失业保险待遇申领办理渠道清单（得分0.60，命中词：咨询） |
| 诊断结果（旧） | ❌ 特征词匹配不足：得分为0 |

### 2.2 根因定位

通过代码追踪，定位到 `diagnose_mismatch` 函数的 **2 个核心缺陷**：

**缺陷1：意图名称匹配方式太粗糙**

原代码使用简单子串包含判断：

```python
benchmark in name or name in benchmark
```

对「了解失业登记」vs「办理失业登记」：

- `"了解失业登记" in "办理失业登记"` → False
- `"办理失业登记" in "了解失业登记"` → False

两者核心词都是「失业登记」，但前缀不同（了解 vs 办理），子串判定失效。

**缺陷2：错误进入后续判定链路**

因 Step 1 误判"标杆不在候选列表"，代码继续向下：

- Step 2：在权重词表中找到「了解失业登记」→ 通过
- Step 3：在 `top_intents` 中查找「了解失业登记」的得分 → 找不到（候选中是「办理失业登记」）→ score=0
- Step 4：输出"特征词匹配不足，得分为0" → **完全误导**

### 2.3 代码执行链路示意

```text
diagnose_mismatch("了解失业登记")
  │
  ├─ Step 1: "了解失业登记" in candidates?
  │   子串匹配 "了解失业登记" vs "办理失业登记" → FAIL ← 缺陷1
  │
  ├─ Step 2: "了解失业登记" in weights? → YES
  │
  ├─ Step 3: top_intents中查找"了解失业登记"的得分
  │   候选列表中只有"办理失业登记" → score=0 ← 缺陷2
  │
  └─ Step 4: 输出"特征词匹配不足" ← 错误结论
```

---

## 三、修复方案

### 3.1 新增 `_find_benchmark_in_set` 辅助函数

实现**三级匹配策略**，替代原来的简单子串判定：

| 优先级 | 策略 | 示例 |
| ------ | ---- | ---- |
| 1 | 完全相等 | `了解失业登记` = `了解失业登记` |
| 2 | 子串包含 | `失业登记` in `了解失业登记` |
| 3 | 去前缀核心词匹配 | `了解`失业登记 ↔ `办理`失业登记 → 核心词相同 ✓ |

常见动词前缀列表：`了解、办理、查询、申请、咨询、申领、领取、获取、查看、知道`

### 3.2 增强诊断细分能力

将原来的 Step 3~4（只查 `top_intents`）细化为三个子步骤：

| 步骤 | 检查内容 | 结论 |
| ---- | -------- | ---- |
| 3a | `top_intents` 中有标杆意图的得分 | topK取得不全（有分但没进候选） |
| 3b | `all_hit_words` 中有标杆意图的命中词 | topK取得不全（命中了词但得分未进榜） |
| 3c | 从 AC 自动机中提取标杆意图的完整特征词列表 | 特征词未命中（列出具体特征词供对照） |

### 3.3 修复后的诊断结果

同一个 Case，修复后正确输出：

```text
大模型筛选问题：标杆意图「了解失业登记」存在于候选列表（得分0.96）但未被AI选中，
AI选择了「查询失业保险待遇申领办理渠道清单」
```

## 六、会议结论

1. 原诊断逻辑的核心问题是**意图名称匹配过于简单**，无法处理"了解X"和"办理X"这类同核心词不同前缀的情况
2. 修复方案通过三级匹配策略（精确→子串→去前缀核心词）解决了名称匹配问题
3. 同时增强了诊断细分能力，从 AC 自动机中直接提取特征词列表，给出更精准的未命中原因
4. 修复后该 Case 正确诊断为"大模型筛选问题"而非"特征词匹配不足"
