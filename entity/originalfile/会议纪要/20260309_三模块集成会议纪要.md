# 会议纪要 — 意图匹配平台三模块集成

**日期**：2026-03-09  
**参会人**：产品方、技术方

---

## 一、会议背景

将意图匹配平台从两模块（特征词管理 + 意图匹配分析）重构为三大功能模块架构。

## 二、核心决策

### 1. 模块划分

| 模块 | 功能 | 数据流 |
|------|------|--------|
| 特征词提取 | 从意图名/匹配结果中AI提取特征词 + 人工补词 | 输出 → 全局词表 |
| 权重分生成 | 基于公式（embedding→IDF×关联度）计算权重 | 读取全局词表，写回权重 |
| 意图匹配定位 | AC自动机匹配 + TOP5大模型筛选 | 读取全局词表进行匹配 |

### 2. 全局词表

- **双格式同步**：JSON + Excel 实时双写
- **列定义**：意图名称、特征词、层级（核心词/发散词/同义词）、来源、时间戳
- **去掉两列**：权重、理由（原AI打分的数据列）
- **公式权重存储**：`公式权重: {意图名: 分值}` 嵌套结构

### 3. 特征词提取优化

- **去掉AI打分步骤**：提取后直接合并到全局词表，不再调用大模型打权重分
- **新增人工补词**：输入意图名+特征词+层级，重复检测，带时间戳备注
- **上传提示列名**：纯意图清单（意图名称）/ 匹配结果清单（原始问题、标杆意图）

### 4. 权重分生成（新模块）

- **冷启动**：全局词表无公式权重时，embedding→cosine sim→IDF×Dirichlet关联度
- **纠偏**：上传原始问-标杆意图清单，用子串匹配命中数据重算权重
- **参考 Demo1**：embedding缓存 + RPM限流重试机制

### 5. 意图匹配定位

- **去掉归一化**：不再做得分归一化处理
- **TOP5策略**：固定取得分前5名意图给大模型筛选（替代原阈值过滤方式）
- **全量定位新格式**：`意图名（命中词列表）(排名)` 格式展示所有命中意图
- **诊断逻辑调整**：将"算法筛选问题"改为"排名超出TOP5"

### 6. 所有页面可下载

- 全局词表可下载 Excel
- 打分结果可下载 Excel
- 匹配结果可下载 Excel

### 7. 文件清理

- 备份 `weight_scorer.py` 和 `run_weight_score.py` 到 `result/备份/`
- 从 `__init__.py` 移除旧模块引用
