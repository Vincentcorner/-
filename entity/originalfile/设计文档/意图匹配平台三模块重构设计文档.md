## 3. 模块一：特征词提取

### 3.1 模式A — 从意图清单提取

- 输入：Excel（列：`意图名称`）
- 处理：AI 生成三维度特征词
- 输出：全局词表

### 3.2 模式B — 从匹配结果提取

- 输入：Excel（列：`原始问题`, `标杆意图`，可选：`转写结果`, `分析结果`）
- 处理：筛选"意图不全/词未命中" → AI 从文本提取特征词
- 输出：全局词表

### 3.3 人工补词

- 输入：意图名称 + 特征词 + 层级
- 校验：重复检测
- 输出：全局词表，备注 = `人工补充@时间`

## 4. 模块二：权重分生成

### 4.1 公式体系（移植自Demo1）

- **F1**: `W(t, I) = 关联度(t, I) × IDF(t)`
- **F2**: `IDF(t) = log₂(N / df(t))`  N=意图总数，df=包含该词的意图数
- **F3**: `关联度(t, I) = (real_count(t→I) + k × sim(t,I)) / (real_count(t→any) + k × Σⱼ sim(t,Iⱼ))`
- **k** = 5（Dirichlet 先验强度系数）

### 4.2 流程

```
冷启动：embedding→sim矩阵→IDF→关联度(real=空)→权重W
纠偏：  上传原始问-标杆→匹配命中→real_counts→重算W
```

### 4.3 embedding策略

- API：硅基流动 `BAAI/bge-large-zh-v1.5`
- 缓存：`embedding_cache.json`，已算过的词不重复调API
- 重试：RPM限流自动等待，指数退避（15s/30s/45s/60s/75s），最多5次
- 进度：SSE实时推送batch进度和重试状态到页面

## 5. 模块三：意图匹配定位

### 5.1 匹配流程

```
原始问 → 三维度转写(AI) → AC自动机匹配 → 原始得分排序
→ TOP5送大模型筛选 → 最终意图
```

### 5.2 TOP5策略

- 不再使用阈值过滤
- 不再做归一化
- AC匹配后所有有得分的意图按原始得分降序排列，取TOP5

### 5.3 全量特征词定位

展示所有命中意图（不限于TOP5），格式：

```
意图名（命中词1、命中词2）(排名)
```

### 5.4 分析结果诊断

| 诊断类别 | 触发条件 |
|----------|----------|
| 一致 | 算法与标杆一致 |
| 大模型筛选问题 | 标杆在TOP5中未被选中 |
| 排名超出TOP5 | 标杆有得分但排名>5 |
| 权重分表意图不全 | 标杆不在词表中 |
| 特征词未命中 | 标杆在词表中但AC未命中 |

## 7. API路由汇总

| `/api/feature-extract` | POST | 提取 | AI提取特征词 |
