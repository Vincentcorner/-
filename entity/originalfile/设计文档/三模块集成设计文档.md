### JSON 格式

```json
{
  "意图映射表": {
    "失业保险金申领": {
      "核心词": ["失业保险金", "申领"],
      "发散词": ["申请", "领取"],
      "同义词": ["失业金"]
    }
  },
  "词权重表": {
    "失业保险金": {
      "来源": "AI提取",
      "时间": "2026-03-09 19:00:00",
      "公式权重": {
        "失业保险金申领": 0.8542,
        "失业保险金标准": 0.3217

    }
  }

```

### Excel 格式（每行一条记录）

| 意图名称 | 特征词 | 层级 | 来源 | 时间 | 公式权重(意图1) | 公式权重(意图2) | ... |

## 3. 模块一：特征词提取

### 输入

- **纯意图清单**：Excel，必需列 `意图名称`
- **匹配结果清单**：Excel，必需列 `原始问题, 标杆意图`

### 处理逻辑

1. 大模型分析意图名/匹配结果，提取三层特征词
2. 自动合并到全局词表（JSON + Excel 双写）
3. 不再做 AI 打权重分

### 人工补词

- 输入：意图名称 + 特征词 + 层级（核心词/发散词/同义词）
- 重复检测：同意图同词同层级重复则提示
- 备注：来源标记"人工补充"，附时间戳

## 4. 模块二：权重分生成

### 算法（移植自 Demo1）

```
W(t, I) = IDF(t) × Relevance(t, I)

IDF(t) = log₂(N / df(t))

Relevance(t, I) = (real(t,I) + k·sim(t,I)) / (Σreal(t) + k·Σsim(t))
```

### 冷启动流程

1. 读取全局词表 → 提取所有词和意图
2. embedding 所有词和意图 → cosine similarity 矩阵
3. 计算 IDF × Dirichlet关联度 → 公式权重
4. 回写到全局词表 `词权重表.公式权重`

### 纠偏流程

1. 上传原始问-标杆意图清单
2. 子串匹配构建 real_counts
3. 重算权重（Dirichlet平滑融合先验+实际命中）
4. 计算 diff，回写全局词表

### 技术特性

- **embedding 缓存**：`result/global/weighted/embedding_cache.json`
- **RPM 限流重试**：自动等待 15s×n 后重试，最多5次
- **进度回调**：SSE 实时展示进度

## 5. 模块三：意图匹配定位

### 匹配流程

```
原始问 ──→ 三维度转写 ──→ 四路AC自动机匹配
                              │
                        全量意图排名（无阈值、无归一化）
                              │
                        取 TOP5 ──→ 大模型筛选
                              │
                        最终意图 + 全量定位展示
```

### TOP5 策略（替代旧阈值方式）

- 不再做归一化计算
- 四路匹配后按原始得分降序排列
- 固定取前5名给大模型筛选
- 仅1个候选时直接取用，不调用大模型

### 全量定位格式

```
失业保险金申领（失业保险金、申领）(1) | 失业登记（失业、登记）(2)
```

展示所有命中意图 + 命中特征词 + 排名。

### 诊断逻辑

| 诊断类型 | 说明 |
|---------|------|
| 大模型筛选问题 | 标杆在TOP5中但AI未选中 |
| 排名超出TOP5 | 标杆有得分但排名>5，大模型看不到 |
| 权重分表意图不全 | 标杆意图不在词表中 |
| 特征词匹配不足 | 转写未覆盖特征词或权重过低 |
