# 需求变更设计文档：意图定位功能优化

> **日期**: 2026-02-23
> **版本**: v1.2
> **变更范围**: Web UI 结果展示区域调整 + 历史记录功能

---

## 1. 变更概述

对意图匹配 Web UI 的结果展示页面进行 4 项调整：

1. **移除置信分列**：该字段为 AI 自我评估分数，不反映实际匹配质量，去除避免误导
2. **意图特征词增加得分显示**：每个意图名后追加 `(得分)` 格式
3. **新增分析结果列**：将诊断分析文字从详情弹窗提升到主表格独立列
4. **历史分析结果查看功能**：可浏览和加载历史批次的分析结果

---

## 2. 置信分分析与移除理由

### 2.1 置信分计算逻辑

置信分存在两条计算路径：

| 路径 | 触发条件 | 计算方式 |
|------|---------|---------|
| AI 筛选 | 多候选意图 + 有筛选提示词 | 大模型返回 `置信度` × 100 |
| 算法兜底 | 单候选或无提示词 | `min(80 + 得分×10, 99)` |

### 2.2 移除原因

- AI 路径的置信度是大模型的「自我评估」，主观性强，无客观校准
- 算法路径的公式是人工拟合，始终偏高（≥80），缺乏区分度
- 该值未参与任何下游决策，展示易造成误解

## 3. 意图特征词格式变更

### 改前

```
办理失业登记（失业登记、查询失业保险待遇）、查询失业保险待遇申领办理渠道清单（咨询）
```

### 改后

```
办理失业登记(0.96)（失业登记、查询失业保险待遇）、查询失业保险待遇申领办理渠道清单(0.60)（咨询）
```

每个意图名后紧跟 `(得分)`（保留2位小数），再跟命中词列表。

---

## 4. 分析结果列

从详情弹窗中的「诊断分析」字段提取，作为独立列直接展示在主表格中：

- 截断显示前50字符，鼠标悬浮可查看全文
- 匹配一致（✓）时绿色显示
- 不一致时橙色显示

---

## 5. 历史分析结果查看

### 5.1 功能说明

在进度区域下方新增可折叠面板「📂 历史分析结果」，支持浏览和加载已有分析批次。

### 5.2 后端 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/history` | GET | 扫描 `result/*/benchmark_compare/` 目录，返回所有历史记录 |
| `/api/history-detail` | GET | 读取指定 JSON 文件内容（带路径安全校验） |

历史记录返回字段：`domain`、`date`、`batch`、`filename`、`total`、`match_count`、`accuracy`

### 5.3 前端交互

- 点击面板标题展开/收起，首次展开时懒加载
- 每条记录显示：领域 / 日期#批次 / 文件名 / 准确率 / 匹配数
- 点击某条记录：加载该批次数据到结果表格及统计区
