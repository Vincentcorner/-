# 诉求转写任务

请将用户的口语化诉求转写为标准化的业务表达。

## 转写规则

1. **保留核心语义**：不改变用户的原始意图
2. **规范化表达**：使用业务标准术语
3. **简化冗余**：去除无关的寒暄语和重复内容
4. **补全省略**：补充必要但被省略的信息
5. **保持简洁**：转写后的表达应简明扼要

## 转写示例

| 用户原始诉求 | 转写后 |
|-------------|--------|
| 我想领失业金 | 失业保险金申领 |
| 公司倒闭了我怎么领钱 | 失业保险金申领 |
| 那个返还的钱怎么查 | 稳岗返还查询 |
| 我不是本地人能领吗 | 失业保险金申领条件查询 |
| 每个月能领多少钱啊 | 失业保险金标准查询 |
| 离职了想办理失业登记 | 失业登记办理 |

## 多意图处理

如果用户诉求可能对应多个意图，用顿号分隔：

| 用户原始诉求 | 转写后 |
|-------------|--------|
| 公司倒闭了怎么办 | 失业保险金申领、失业登记 |
| 我想查我的失业保险缴费和领取情况 | 失业保险缴费查询、失业保险金领取记录查询 |

## 输出格式

直接输出转写后的标准表达，不需要解释。

如有多个可能的意图，用顿号（、）分隔。
