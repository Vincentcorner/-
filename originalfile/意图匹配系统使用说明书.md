## 1. 系统概述

本系统用于将用户的自然语言诉求（如"我失业了怎么领失业金"）**自动匹配到预定义的业务意图**（如"失业保险金申领"）。

**核心流程**：

```
上传问题 Excel → 三维度 AI 转写 → AC 自动机匹配 → 大模型意图筛选 → 结果展示/下载
```

系统提供两种使用方式：**Web UI 可视化操作**（推荐）和**命令行批量执行**。

## 2. 前置准备

### 2.1 环境依赖

#### Python 环境

- Python 3.10+
- 安装依赖：

```powershell
pip install flask pandas openpyxl pyahocorasick dashscope
```

#### API 配置

系统使用**通义千问 Qwen 32B** 大模型 API 进行转写和意图筛选。需要确保：

- 已配置有效的 DashScope API Key（通过环境变量 `DASHSCOPE_API_KEY` 或在代码中设置）
- API 调用间隔默认 **5 秒/次**（可在 `scripts/common/llm_api.py` 中调整 `DEFAULT_INTERVAL`）

### 2.2 配置文件

所有配置文件位于 `config/` 目录下：

#### ① `threshold.json` — 意图筛选阈值

```json
{
    "min_score": 0.4,
    "top_k": 5
}
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `min_score` | 意图得分最低阈值，低于此分数的意图会被过滤 | 0.4 |
| `top_k` | 最多保留的候选意图数量 | 5 |

> **调整建议**：如果测试中发现漏筛（正确意图被过滤），可降低 `min_score`；如果噪音太多，可提高。

#### ② `layer_weights.json` — 特征词层级权重

```json
{
    "L1": 1.0,
    "L2": 0.8,
    "L3": 0.6
}
```

| 层级 | 含义 | 默认权重 |
|------|------|---------|
| L1 | 事项词（核心业务关键词） | 1.0 |
| L2 | 动作词（业务动作关键词） | 0.8 |
| L3 | 情境词（场景辅助关键词） | 0.6 |

> 得分计算公式：`意图得分 = Σ(层级权重 × 词自身权重)`

#### ③ `domains.json` — 领域配置

```json
{
    "失业保险": {
        "name": "失业保险",
        "description": "失业保险相关业务",
        "keywords": ["失业", "失业金", "失业保险", "稳岗", "技能补贴"]
    }
}
```

> 新增领域时，需要在此文件添加对应条目。

### 2.3 数据文件

#### ① 标杆问题 Excel（必需）

- **位置**：`originalfile/{领域}/深圳原始数据（部分）.xlsx`（或自定义 Excel）
- **必需列**：`原始问`（包含待匹配的用户问题）
- **可选列**：`意图`（标杆意图，用于自动对比准确率）
- **格式要求**：`.xlsx` 或 `.xls`

示例格式：

| 原始问 | 意图 |
|--------|------|
| 我失业了怎么领取失业保险金 | 失业保险金申领 |
| 失业补助金在哪里申请 | 失业补助金申领 |

#### ② 权重词表（必需）

- **位置**：`result/{领域}/weighted/weighted_words.json`
- **来源**：通过 `/wordlist` 工作流生成
- **内容**：包含 `意图映射表` 和 `词权重表` 两个字段

> ⚠️ **如果没有权重词表，系统无法运行**。请先通过 `/wordlist` 工作流生成。

### 2.4 提示词文件

提示词文件位于 `prompts/` 目录下，系统会自动加载：

| 文件 | 用途 | 是否必需 |
|------|------|---------|
| `query_rewrite_3d_prompt.md` | 三维度转写提示词（情形/群众语言/官方表达） | ✅ 必需 |
| `intent_select_prompt.md` | AI 意图筛选提示词 | 可选（缺失时退化为 Top-1） |

> 这些提示词文件通常无需修改，除非需要调整转写或筛选策略。

## 3. 使用流程

### 3.1 Web UI 方式（推荐）

#### 步骤 1：启动服务器

```powershell
py scripts/web_app.py
```

启动成功后会输出：

```
==================================================
  意图匹配 Web UI
  项目根目录: d:\...\entity
  访问地址:   http://localhost:5000
==================================================
```

#### 步骤 2：打开浏览器

访问 **<http://localhost:5000**，进入「🎯> 意图匹配分析平台」页面。

#### 步骤 3：配置参数

在页面上操作以下配置项：

| 配置项 | 说明 | 操作 |
|--------|------|------|
| **上传文件** | 拖拽或点击选择 Excel 文件 | 支持 `.xlsx` / `.xls` |
| **领域** | 下拉选择业务领域 | 带 ✓ 标记的领域表示已有权重词表 |
| **起始行号** | 从 Excel 第几行数据开始 | 默认 1（第一行数据），用于分批测试 |
| **分析条数** | 本次处理多少条 | 默认 20，最大 500 |
| **权重词表** | 自动检测或手动上传 | 默认自动检测领域下的词表 |

#### 步骤 4：开始分析

点击 **🚀 开始分析** 按钮，进入执行流程：

1. **转写阶段**：逐条调用 AI 进行三维度转写（页面显示实时进度条）
2. **匹配阶段**：AC 自动机匹配 + AI 意图筛选（结果逐条实时展示在表格中）
3. **完成**：显示统计卡片（总条数 / 匹配一致 / 不一致 / 准确率）

#### 步骤 5：查看与下载结果

- **页面标题**：结果区域上方显示「🔍 人工分类校验问题自动化」标题
- **流程图**：标题下方展示自动化校验流程示意图（诊断分析判定逻辑）
- **页面表格**：实时展示每条数据的匹配结果
- **📋 详情按钮**：每行末尾可点击查看详细诊断分析（含：诊断原因、全量意图得分明细、命中特征词等）
- **📥 下载 Excel**：点击可下载完整结果文件

详情弹窗中的**诊断分析**会自动判定匹配不一致的原因：

| 诊断类型 | 含义 |
|----------|------|
| ✓ 一致 | 算法意图与标杆意图匹配 |
| 大模型筛选问题 | 标杆意图在候选列表中但AI未选中 |
| 权重分表意图不全 | 标杆意图不在权重词表中 |
| topK意图取得不全 | 标杆意图有得分但未进入候选列表 |
| 特征词匹配不足 | 特征词未覆盖或权重分过低 |

### 3.2 命令行方式

#### 全自动模式（推荐）

```powershell
py scripts/batch_intent_match.py -d 失业保险 -c 20 --auto-rewrite
```

#### 使用已有转写文件（跳过 API 转写）

```powershell
py scripts/batch_intent_match.py -d 失业保险 -c 20 -r result/失业保险/benchmark_compare/20260211/auto_rewrite_3d.json
```

#### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-d / --domain` | 领域名称 | 失业保险 |
| `-c / --count` | 处理数量 | 50 |
| `-f / --file` | 问题 Excel 文件路径 | 标杆数据 |
| `-w / --weights` | 权重词表路径 | 自动检测 |
| `-r / --rewrite` | 已有转写结果 JSON | 无 |
| `--auto-rewrite` | 自动调用 API 转写 | 关闭 |

### 3.3 分批测试操作

#### Web UI 分批测试（推荐）

同一个 Excel 文件无需拆分，直接通过**起始行号 + 分析条数**控制每批范围：

| 批次 | 起始行号 | 分析条数 | 实际处理范围 |
|------|---------|---------|-------------|
| 第 1 批 | 1 | 20 | 第 1~20 条 |
| 第 2 批 | 21 | 20 | 第 21~40 条 |
| 第 3 批 | 41 | 20 | 第 41~60 条 |
| 第 4 批 | 61 | 20 | 第 61~80 条 |

**操作步骤**：

1. 上传 Excel → 设置起始行号和条数 → 点击"开始分析"
2. 等待完成 → 下载 Excel 结果
3. **修改起始行号**（如 21）→ 再次点击"开始分析"
4. 重复以上步骤直到测试完毕

> 每批结果**独立保存**在不同的输出目录中，不会互相覆盖。

#### 时间预估

| 条数 | 预估耗时 | 说明 |
|------|---------|------|
| 10 条 | 2~3 分钟 | 快速验证 |
| 20 条 | 4~6 分钟 | 常规批次 |
| 50 条 | 10~15 分钟 | 较大批次 |

> 耗时主要来自 API 调用间隔（每条约 5 秒转写 + 可能的 5 秒 AI 筛选）。

## 4. 核心算法说明

### 4.1 三维度 AI 转写

将用户原始问题转写为 3 个语义维度：

| 维度 | 英文标识 | 说明 | 示例 |
|------|---------|------|------|
| 情形提取 | `scenario` | 用户面临的具体情境 | 在深圳工作，被公司辞退 |
| 群众语言 | `plain_language` | 口语化的表达方式 | 失业了想领失业金 |
| 官方表达 | `official_expression` | 政策规范用语 | 申领失业保险金 |

### 4.2 AC 自动机匹配

对每个维度的转写文本独立执行多模式匹配：

1. **AC 自动机扫描**：在文本中查找所有已知特征词（一次扫描，复杂度 O(n)）
2. **子串过滤**：重叠命中时保留最长匹配（如同时命中"保险金"和"失业保险金"，只保留后者）
3. **负面清单过滤**：排除指定噪音词

### 4.3 得分计算

```
意图得分 = Σ(层级权重 × 词自身权重)
```

- 三个维度分别独立计算，同一意图取**最高分**
- 得分 ≥ `min_score`（默认 0.4）的意图保留为候选
- 按得分降序，最多保留 `top_k`（默认 5）个候选

### 4.4 AI 意图筛选

| 候选数量 | 处理方式 |
|---------|---------|
| 0 个 | 输出 `(无匹配)` |
| 1 个 | 直接取该意图 |
| ≥ 2 个 | 调用大模型从候选中选出最终意图 |

### 4.5 置信分

| 场景 | 计算方式 | 范围 |
|------|---------|------|
| 无匹配 | 0 | 0 |
| AI 筛选 | 大模型返回的置信度 × 100 | 0~99 |
| 算法 Top-1 | min(80 + 得分 × 10, 99) | 80~99 |

页面颜色分级：🟢 ≥ 80（高） / 🟡 ≥ 50（中） / 🔴 < 50（低）

## 5. 产物说明

### 5.1 输出目录与文件命名

```
result/{领域}/benchmark_compare/{日期}/{批次}/
├── {源文件名}_R{起始行}_C{条数}.json    # 详细结果（JSON 格式）
└── {源文件名}_R{起始行}_C{条数}.xlsx    # 结果报表（Excel 格式）
```

**命名规则**：`{上传的Excel文件名}_R{起始行号}_C{实际处理条数}`

**示例**：上传 `深圳原始数据（部分）.xlsx`，起始行号 21，分析 20 条 →

```
深圳原始数据（部分）_R21_C20.xlsx
深圳原始数据（部分）_R21_C20.json
```

- **日期**：格式 `YYYYMMDD`，如 `20260223`
- **批次**：自动递增编号，如 `batch_01`、`batch_02`

### 5.2 输出字段说明

Excel 结果文件包含以下列：

| 列名 | 说明 |
|------|------|
| **序号** | 数据编号 |
| **原始问题** | 用户原始诉求文本（截取前 80 字符） |
| **转写结果** | 三维度转写内容，格式：`[情形] ... | [群众语言] ... | [官方表达] ...` |
| **转写来源** | `AI三维度`（转写成功）或 `原始`（转写失败时用原文匹配） |
| **算法意图** | 系统最终匹配的意图名称 |
| **全量意图** | 所有候选意图（用"、"分隔） |
| **意图特征词** | 每个候选意图命中的特征词，格式：`意图名（词1、词2）` |
| **置信分** | 匹配置信度（0~99），越高越可信 |
| **标杆意图** | Excel 原文中的标杆意图（用于对比） |
| **是否一致** | `✓`（算法意图与标杆一致）或 `✗`（不一致） |
| **详情** | 包含诊断分析、完整问题、全量意图得分、命中词等（点击弹窗查看） |

### 5.3 统计指标

页面和结果文件中提供以下统计：

| 指标 | 说明 |
|------|------|
| **总条数** | 本批次处理的数据总量 |
| **匹配一致** | 算法意图与标杆意图一致的条数 |
| **不一致** | 算法意图与标杆不一致的条数 |
| **准确率** | 一致条数 / 总条数 × 100% |

> **一致判定规则**：算法意图包含标杆意图文本，或标杆意图包含算法意图文本（模糊匹配）。

## 6. 常见问题

### Q1：权重词表从哪里来？

通过 `/wordlist` 工作流生成，命令：

```powershell
py scripts/workflows/run_wordlist.py -d 失业保险 --auto
```

生成后保存在 `result/失业保险/weighted/weighted_words.json`。

### Q2：如何新增一个领域？

1. 在 `config/domains.json` 中添加领域条目
2. 在 `originalfile/{新领域}/` 下放置标杆问题 Excel
3. 运行 `/wordlist` 工作流生成该领域的权重词表
4. 即可在 Web UI 或命令行中使用新领域

### Q3：API 调用报错怎么办？

- 检查 API Key 是否有效
- 检查网络连接
- 确认 API 余额是否充足
- 查看终端日志中的详细错误信息

### Q4：转写失败的条目如何处理？

转写失败的条目会自动降级为"原始"匹配模式，使用原始文本直接进行 AC 匹配。结果中 `转写来源` 列会标记为 `原始`。

### Q5：如何调整匹配精度？

| 调整方向 | 操作 |
|---------|------|
| 提高召回（减少漏筛） | 降低 `config/threshold.json` 中 `min_score` |
| 提高精度（减少误匹配） | 提高 `min_score`，或调整权重词表质量 |
| 增加候选意图数量 | 调大 `top_k` |
| 调整层级重要性 | 修改 `config/layer_weights.json` 中各层级权重 |
