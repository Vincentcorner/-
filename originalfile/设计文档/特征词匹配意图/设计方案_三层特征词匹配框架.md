## 一、背景与问题

### 1.1 现有方案的痛点

当前意图识别流程采用多级串行架构：

```
用户问题 → 领域识别 → 问题类型 → 向量匹配 → 模型精选 → 意图
```

**主要问题**：
1. **慢**：多次模型调用 + 向量检索，延迟累积
2. **准确率递减**：每步90%准确 → 四步后 ≈ 65%（级联错误）
3. **特征词穷举困难**：动作词随客体变化，无法穷举

### 1.2 目标

设计一个基于特征词的快速匹配框架：
- **快**：优先使用规则匹配，减少模型调用
- **准**：直接定位意图候选集，避免级联错误
- **可维护**：分层设计，词表职责清晰

## 二、核心设计：三层认知模型

### 2.1 用户表达的三层结构

用户的问题表达与其对业务的理解程度相关，可分为三层：

| 层级 | 名称 | 特点 | 用户表达示例 | 匹配难度 |
|------|------|------|--------------|----------|
| L1 | **事项层** | 直接说业务名称 | "办理营业执照变更" | ⭐ 简单 |
| L2 | **诉求层** | 表达需求/目的 | "我想开公司" | ⭐⭐ 中等 |
| L3 | **情景层** | 描述当前情况 | "经理辞职了" | ⭐⭐⭐ 困难 |

### 2.2 与实体命中策略的词典体系整合

> 参考：[会议纪要_实体命中策略讨论_第三轮_用户表达层](../03_模块三_实体命中/会议纪要/会议纪要_实体命中策略讨论_第三轮_用户表达层_20260126.md)

本框架与之前讨论的**三词典体系**形成对应关系：

| 本框架层级 | 对应词典 | 词典定义 | 典型表达 |
|-----------|----------|----------|----------|
| **L1 事项层** | 俗称词典 | 用户对业务概念的口语表达 | "失业金"→"失业保险金" |
| **L2 诉求层** | 通用场景词典 | 用户所处的客观情境 | "我刚离职"、"我买房了" |
| **L3 情景层** | 用户状态词典 | 用户描述自己的主观状态 | "我好穷"、"我很迷茫" |

**整合后的完整词典体系**：

│  └─────────────────┘  └─────────────────┘                   │

### 2.4 匹配策略：并行匹配 + 加权得分

> **核心改进**：不再逐级匹配，而是三层同时匹配，综合得分排序

```
┌─────────────────────────────────────────────────────────────┐
│                        用户问题                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    并行匹配三层特征词                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐          │
│  │ L1 事项层    │  │ L2 诉求层    │  │ L3 情景层    │          │
│  │ 权重: 1.0   │  │ 权重: 0.8   │  │ 权重: 0.6   │          │
│  └─────────────┘  └─────────────┘  └─────────────┘          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                       意图得分汇总                           │
│  - 收集三层命中的所有意图                                     │
│  - 每个意图的得分 = Σ(层级权重 × 词权重 × 命中次数)            │
│  - 应用前提条件调整系数                                       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                       候选意图输出                           │
│  - 按得分排序，取Top-K作为候选                                │
│  - 如果最高分 ≥ 阈值：高置信候选集                            │
│  - 如果最高分 < 阈值：低置信，需模型精选                       │
│  - 如果无命中：模型兜底                                       │
└─────────────────────────────────────────────────────────────┘
```

**并行匹配的优势**：

| 对比项 | 逐级匹配 | 并行匹配+加权 |
|--------|----------|---------------|
| 级联错误 | L1错 → 全错 | 多层信号互补，容错性高 |
| 信息利用 | 只用一层信息 | 综合三层信息 |
| 复杂表达 | 难以处理混合表达 | 自然处理"经理辞职想变更" |
| 置信度 | 二元（命中/未命中）| 连续得分，可排序 |

#### 3.2.1 L1 词表来源与管理

| 来源类型 | 说明 | 权重建议 | 管理方式 |
|----------|------|----------|----------|
| **系统生成** | 意图名称本身 | 1.0 | 随意图库自动同步 |
| **Excel同义表达** | 从特征词Excel导入 | 0.85-0.95 | 定期从Excel同步 |
| **人工补充** | 运营人员手动添加 | 0.8-0.95 | 通过管理界面维护 |
| **日志挖掘** | 从未命中日志中发现 | 0.7-0.9 | 需人工审核后入库 |

**权重设置原则**：
- **1.0**：完全匹配业务标准术语（意图名称）
- **0.9-0.95**：官方同义表达（如"年检"="年报报送"）
- **0.8-0.9**：口语化表达（如"开公司"="企业开办"）
- **0.7-0.8**：模糊或宽泛表达（需谨慎，可能有歧义）

#### 3.3.1 L2 词表来源与管理

| 来源类型 | 说明 | 权重建议 | 管理方式 |
|----------|------|----------|----------|
| **人工整理** | 业务专家根据经验整理 | 0.8-0.95 | 定期review并更新 |
| **日志挖掘** | 从用户问题日志中提取高频诉求 | 0.6-0.85 | 需人工审核 |
| **用户反馈** | 用户反馈"我其实想xxx"类表达 | 0.7-0.9 | 通过反馈渠道收集 |

**诉求词与意图的关联权重**：
- **1.0**：诉求词与意图强关联（如"公司搬家"→"变更经营场所"）
- **0.7-0.9**：诉求词与意图中等关联（如"创业"→"企业开办"）
- **0.5-0.7**：诉求词与意图弱关联，作为补充候选

**得分计算**：
```
意图得分 = 层级权重(0.8) × 诉求词权重 × 关联权重
```

## 四、匹配算法

### 4.1 核心算法：并行匹配 + 加权得分

```python
from dataclasses import dataclass, field
from typing import List, Dict
from collections import defaultdict

@dataclass
class IntentScore:
    """意图得分记录"""
    intent_name: str
    total_score: float = 0.0
    hit_details: List[Dict] = field(default_factory=list)  # 命中详情

@dataclass
class MatchResult:
    """匹配结果"""
    candidates: List[IntentScore]      # 候选意图（按得分排序）
    top_score: float                   # 最高分
    confidence_level: str              # "high" / "medium" / "low" / "fallback"
    need_model: bool                   # 是否需要模型精选


def match_intent(user_query: str, config: Dict) -> MatchResult:
    """
    三层并行匹配主流程
    """
    # 预处理
    query = preprocess(user_query)  # 分词、纠错等
    
    # 意图得分表
    intent_scores: Dict[str, IntentScore] = defaultdict(
        lambda: IntentScore(intent_name="")
    )
    
    # ========== 并行匹配三层 ==========
    
    # L1 事项层匹配
    l1_hits = match_l1_matter(query)
    for hit in l1_hits:
        intent = hit["intent"]
        score = config["层级权重"]["L1_事项层"] * hit["word_weight"]
        
        if intent not in intent_scores:
            intent_scores[intent] = IntentScore(intent_name=intent)
        intent_scores[intent].total_score += score
        intent_scores[intent].hit_details.append({
            "layer": "L1", "word": hit["word"], "score": score
        })
    
    # L2 诉求层匹配
    l2_hits = match_l2_demand(query)
    for hit in l2_hits:
        for candidate in hit["candidates"]:
            intent = candidate["intent"]
            score = (config["层级权重"]["L2_诉求层"] * 
                     hit["demand_weight"] * 
                     candidate["relation_weight"])
            
            if intent not in intent_scores:
                intent_scores[intent] = IntentScore(intent_name=intent)
            intent_scores[intent].total_score += score
            intent_scores[intent].hit_details.append({
                "layer": "L2", "word": hit["demand_word"], "score": score
            })
    
    # L3 情景层匹配
    l3_hits = match_l3_scene(query)
    for hit in l3_hits:
        for intent in hit["intent_candidates"]:
            score = (config["层级权重"]["L3_情景层"] * 
                     hit["object_weight"] * 
                     hit["action_weight"])
            
            if intent not in intent_scores:
                intent_scores[intent] = IntentScore(intent_name=intent)
            intent_scores[intent].total_score += score
            intent_scores[intent].hit_details.append({
                "layer": "L3", 
                "object": hit["object"], 
                "action": hit["action"],
                "score": score
            })
    
    # ========== 应用前提条件调整 ==========
    apply_precondition_adjustments(query, intent_scores)
    
    # ========== 排序并输出结果 ==========
    sorted_intents = sorted(
        intent_scores.values(), 
        key=lambda x: x.total_score, 
        reverse=True
    )[:config["阈值配置"]["候选数量上限"]]
    
    if not sorted_intents:
        return MatchResult(
            candidates=[],
            top_score=0,
            confidence_level="fallback",
            need_model=True
        )
    
    top_score = sorted_intents[0].total_score
    
    # 判断置信度级别
    if top_score >= config["阈值配置"]["高置信阈值"]:
        confidence_level = "high"
        need_model = False  # 高置信可直接返回
    elif top_score >= config["阈值配置"]["低置信阈值"]:
        confidence_level = "medium"
        need_model = True   # 中置信需模型确认
    else:
        confidence_level = "low"
        need_model = True   # 低置信需模型精选
    
    return MatchResult(
        candidates=sorted_intents,
        top_score=top_score,
        confidence_level=confidence_level,
        need_model=need_model
    )


def match_l1_matter(query: str) -> List[Dict]:
    """L1 事项层：匹配业务名称，返回所有命中"""
    hits = []
    for item in 事项词表:
        for word_info in item.事项词:
            if word_info["词"] in query:
                hits.append({
                    "intent": item.意图名称,
                    "word": word_info["词"],
                    "word_weight": word_info["权重"]
                })
    return hits


def match_l2_demand(query: str) -> List[Dict]:
    """L2 诉求层：匹配需求表达，返回所有命中"""
    hits = []
    for item in 诉求词表:
        if item.诉求词 in query:
            hits.append({
                "demand_word": item.诉求词,
                "demand_weight": item.权重,
                "candidates": [
                    {"intent": c.意图, "relation_weight": c.关联权重}
                    for c in item.意图候选
                ]
            })
    return hits


def match_l3_scene(query: str) -> List[Dict]:
    """L3 情景层：客体+情景动作组合匹配"""
    hits = []
    
    # 1. 识别客体
    detected_objects = []
    for obj in 客体词库:
        for syn in obj.同义词:
            if syn["词"] in query:
                detected_objects.append({
                    "object": obj.标准名称,
                    "category": obj.所属类别,
                    "base_weight": obj.基础权重,
                    "word_weight": syn["权重"]
                })
                break
    
    # 2. 对每个客体，识别情景动作
    for obj_info in detected_objects:
        category = obj_info["category"]
        action_dict = 情景动作词库.get(category + "类", {})
        action_dict.update(情景动作词库.get("通用", {}))
        
        for action_type, action_info in action_dict.items():
            for scene_word in action_info.情景动作词:
                if scene_word["词"] in query:
                    # 查找意图映射
                    for mapping in 客体动作映射:
                        if (mapping.客体 == obj_info["object"] and 
                            mapping.动作 == action_info.标准动作):
                            hits.append({
                                "object": obj_info["object"],
                                "action": action_info.标准动作,
                                "object_weight": obj_info["base_weight"] * obj_info["word_weight"],
                                "action_weight": scene_word["权重"],
                                "intent_candidates": mapping.意图候选
                            })
    
    return hits


def apply_precondition_adjustments(query: str, intent_scores: Dict):
    """应用前提条件调整得分"""
    for precond in 前提条件表:
        for word in precond.前提词:
            if word in query:
                for impact in precond.影响意图:
                    intent = impact.原意图
                    if intent in intent_scores:
                        multiplier = impact.get("提升权重", 1.0)
                        if "降低权重" in impact:
                            multiplier = impact["降低权重"]
                        intent_scores[intent].total_score *= multiplier
                        intent_scores[intent].hit_details.append({
                            "layer": "前提条件",
                            "word": word,
                            "multiplier": multiplier
                        })
```

### 4.2 得分计算公式

```
意图总得分 = Σ(各层命中得分) × 前提条件调整系数

其中：
- L1命中得分 = 层级权重(1.0) × 词权重
- L2命中得分 = 层级权重(0.8) × 诉求词权重 × 关联权重
- L3命中得分 = 层级权重(0.6) × 客体基础权重 × 同义词权重 × 动作词权重

前提条件调整系数：
- 命中增强前提 → ×1.5 或 ×2.0
- 命中削弱前提 → ×0.5
```

### 4.3 置信度判定规则

| 得分区间 | 置信度 | 处理方式 |
|----------|--------|----------|
| ≥ 0.8 | 高 | 直接返回Top-1，无需模型 |
| 0.4 - 0.8 | 中 | 返回候选集，模型精选 |
| < 0.4 | 低 | 返回候选集，模型精选（需警示） |
| 无命中 | 兜底 | 全量交给模型处理 |

### 5.4 提问维度与问题类型映射

用户提问的维度（疑问词）会影响问题的分类，同一业务词在不同维度下指向不同问题类型：

| 提问维度 | 疑问词特征 | 可能的问题类型 |
|----------|------------|----------------|
| **How（方式）** | 怎么、如何、怎样 | 软件操作指导、办事渠道、业务规则、测算数值、查询信息 |
| **Where（何地）** | 哪里、在哪、去哪 | 办事渠道、服务场所、软件操作指导 |
| **What（什么）** | 什么、什么是 | 术语解释、业务规则、信息查询 |
| **When（何时）** | 什么时候、多久、几点 | 业务规则、服务场所明细 |
| **Why（为什么）** | 为什么、为啥 | 业务规则、软件操作指导 |
| **Whether（是否）** | 能不能、可不可以 | 业务规则、软件操作指导 |

## 六、示例演示

### 6.1 单层命中示例

```
用户: "我想办理营业执照变更"

并行匹配:
  L1: "营业执照变更" 命中 → 得分 = 1.0 × 0.95 = 0.95
  L2: 未命中
  L3: "营业执照" 命中客体 → 但无动作词

得分汇总:
  意图           得分    来源
  ─────────────────────────────
  营业执照变更   0.95    L1

结果:
  Top-1: 营业执照变更 (0.95)
  置信度: 高
  需要模型: 否
```

### 6.2 多层叠加示例（核心优势）

```
用户: "经理辞职了想变更一下"

并行匹配:
  L1: "变更" 命中多个事项词（模糊匹配，权重较低 0.5）
      - 变更经理: 0.5
      - 变更股东: 0.5
      - 变更经营范围: 0.5
      ...
  L2: 未命中
  L3: "经理"(客体) + "辞职"(情景动作) 命中
      - 变更经理: 0.6 × 0.9 × 1.0 × 0.9 = 0.486

得分汇总（多层叠加）:
  意图           L1得分   L3得分   总得分
  ──────────────────────────────────────
  变更经理       0.50     0.486    0.986  ← 多层叠加，得分最高
  变更股东       0.50     0        0.50
  变更经营范围   0.50     0        0.50
  ...

结果:
  Top-1: 变更经理 (0.986)
  置信度: 高
  需要模型: 否
  
说明: 虽然L1层多个意图都有命中，但L3层的"经理+辞职"组合
      只给"变更经理"加分，因此它最终得分最高
```

### 6.3 混合表达示例

```
用户: "我想开公司，经营范围要办餐饮"

并行匹配:
  L1: 未命中完整事项词
  L2: "开公司" 命中
      - 企业开办: 0.8 × 0.9 × 1.0 = 0.72
      - 个体工商户设立: 0.8 × 0.9 × 0.7 = 0.504
  L3: "经营范围" 命中客体 + "办" 命中通用动作
      - 变更经营范围: 0.6 × 0.85 × 0.9 × 0.8 = 0.367

得分汇总:
  意图               得分
  ───────────────────────
  企业开办           0.72
  个体工商户设立     0.504
  变更经营范围       0.367

结果:
  候选: [企业开办, 个体工商户设立, 变更经营范围]
  Top-1: 企业开办 (0.72)
  置信度: 中
  需要模型: 是（识别到用户想开公司且关注经营范围，需确认）
```

### 6.4 情景动作按类别区分

```
用户A: "经理罢免怎么办"
用户B: "经营场所搬迁怎么办"

匹配过程A:
  L3: 客体"经理"(人员类) + 动作"罢免"(人员类.变更)
      → 变更经理: 0.6 × 0.9 × 1.0 × 0.85 = 0.459

匹配过程B:
  L3: 客体"经营场所"(场所类) + 动作"搬迁"(场所类.变更)
      → 变更经营场所: 0.6 × 0.85 × 1.0 × 0.9 = 0.459

关键点: 
  - "罢免"只在人员类客体下生效
  - "搬迁"只在场所类客体下生效
  - 避免了"经理搬迁"这种错误组合
```

### 6.5 前提条件影响

```
用户: "执照被吊销了想注销"

匹配过程:
  L3: 
    客体: 营业执照
    情景动作: "注销" → 标准动作=注销
    意图候选: [市场主体注销]
  前提条件调整:
    检测到"吊销" → 提升"市场主体注销"权重

结果:
  意图: [市场主体注销]
  置信度: 0.85 (因有前提支撑)
```

### 9.3 词典质量保障

#### 9.3.1 权重管理规范

| 权重范围 | 适用场景 | 示例 |
|----------|----------|------|
| **0.9 ~ 1.0** | 精确匹配，无歧义 | 意图名称本身、官方同义词 |
| **0.7 ~ 0.9** | 高可信口语表达 | "年检"→年报报送 |
| **0.5 ~ 0.7** | 有一定歧义 | "变更"→多个变更类意图 |
| **< 0.5** | 不建议添加 | 歧义太大，宁可不匹配 |

#### 9.3.2 新词入库流程

```
发现新表达 → 初评（是否有歧义）→ 设定权重 → 测试验证 → 正式入库
     │              │                │           │
     │              ↓                ↓           ↓
     │         歧义大：不入库    权重0.7~0.9   确认无误
     │         歧义小：继续                    则上线
     └─────────────────────────────────────────────────
```
